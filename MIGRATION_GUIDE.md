# Руководство по миграции продуктов

## Обзор

Система миграции продуктов позволяет автоматически диагностировать и копировать старые продукты в общую библиотеку. Все операции выполняются через веб-интерфейс в админ-панели.

## Доступ

1. Войдите в систему как администратор
2. Перейдите в **Dashboard → Администрирование**
3. Нажмите кнопку **"Миграция продуктов"** или перейдите по ссылке `/dashboard/admin/products-migration`

## Процесс миграции

### Шаг 1: Диагностика

Нажмите кнопку **"Запустить диагностику"** для анализа текущего состояния продуктов в базе данных.

Диагностика покажет:
- **Всего продуктов** - общее количество в базе
- **В библиотеке** - продукты с `isInLibrary: true` и без `barId`
- **Старые (кандидаты)** - продукты без `barId` и без `isInLibrary: true` (готовы к копированию)
- **Персональные** - продукты с `barId` (не копируются)
- **Премиксы** - продукты с `category === 'Premix'` или `isPremix === true`
- **Аномалии** - продукты с `barId` и `isInLibrary: true` одновременно

### Шаг 2: Миграция

После диагностики нажмите **"Запустить миграцию"** для копирования старых продуктов в библиотеку.

Миграция:
- Находит все продукты без `barId` и без `isInLibrary: true`
- Устанавливает `isInLibrary: true` для этих продуктов
- **Сохраняет оригинальные ID** продуктов (ссылки в других коллекциях не нарушаются)
- Пропускает премиксы, персональные продукты и продукты уже в библиотеке

### Шаг 3: Проверка результатов

После миграции запустите диагностику снова, чтобы убедиться, что продукты успешно скопированы.

## Технические детали

### API Endpoints

- `GET /api/admin/diagnose-products` - диагностика состояния продуктов
- `POST /api/admin/migrate-products` - копирование продуктов в библиотеку

### Переменные окружения

Для работы API endpoints требуется переменная окружения:
- `FIREBASE_SERVICE_ACCOUNT_KEY` - JSON-строка с credentials Firebase Admin SDK

Эта переменная должна быть установлена в Vercel:
1. Перейдите в настройки проекта в Vercel
2. Откройте раздел **Environment Variables**
3. Добавьте `FIREBASE_SERVICE_ACCOUNT_KEY` со значением JSON из service account

### Безопасность

- API endpoints доступны только на сервере
- Для получения доступа к странице миграции требуется роль администратора
- Все операции логируются на сервере

## Часто задаваемые вопросы

### Почему некоторые продукты не копируются?

Продукты не копируются если:
- Они являются премиксами (`category === 'Premix'` или `isPremix === true`)
- Они имеют `barId` (это персональные продукты пользователей)
- Они уже имеют `isInLibrary: true`

### Сохраняются ли ID продуктов после миграции?

Да, ID продуктов сохраняются. Миграция использует `set()` с `merge: true`, что обновляет существующие документы, сохраняя их ID. Это гарантирует, что ссылки в других коллекциях (например, `InventoryLine`, `PurchaseOrderLine`) продолжают работать.

### Можно ли откатить миграцию?

Технически можно, но это требует ручного удаления поля `isInLibrary` для скопированных продуктов. Рекомендуется делать резервную копию базы данных перед миграцией.

### Сколько времени занимает миграция?

Время зависит от количества продуктов. Миграция выполняется батчами по 500 продуктов, поэтому для больших баз данных это может занять несколько секунд или минут.

## Логирование

Все операции логируются на сервере. В случае ошибок проверьте логи Vercel или консоль браузера (для диагностики).
