# Скрипты миграции и диагностики продуктов

## Требования

Для запуска скриптов требуется:
1. **Firebase Admin SDK** - установлен через `npm install firebase-admin`
2. **Учетные данные Firebase Admin** - один из вариантов:
   - Файл `serviceAccountKey.json` в корне проекта
   - Переменная окружения `FIREBASE_SERVICE_ACCOUNT_KEY` с JSON-содержимым service account

## Получение service account key

1. Откройте [Firebase Console](https://console.firebase.google.com/)
2. Выберите ваш проект
3. Перейдите в **Project Settings** → **Service Accounts**
4. Нажмите **Generate New Private Key**
5. Сохраните JSON-файл как `serviceAccountKey.json` в корне проекта

**⚠️ ВАЖНО:** Не коммитьте `serviceAccountKey.json` в Git! Добавьте его в `.gitignore`.

## Скрипты

### 1. Диагностика продуктов (`diagnose-products.ts`)

Анализирует состояние продуктов в базе данных и показывает статистику.

**Запуск:**
```bash
npx tsx scripts/diagnose-products.ts
```

**Что показывает:**
- Общее количество продуктов
- Количество премиксов
- Количество продуктов в библиотеке
- Количество персональных продуктов (с barId)
- Количество старых продуктов (кандидатов для копирования)
- Примеры продуктов каждой категории

### 2. Копирование старых продуктов в библиотеку (`copy-old-products-to-library.ts`)

Безопасно копирует старые продукты (без `barId` и без `isInLibrary: true`) в библиотеку, устанавливая `isInLibrary: true`.

**Запуск:**
```bash
npx tsx scripts/copy-old-products-to-library.ts
```

**Что делает:**
- Находит все продукты без `barId` и без `isInLibrary: true`
- Пропускает премиксы
- Пропускает персональные продукты (с `barId`)
- Пропускает продукты уже в библиотеке
- Обновляет оставшиеся продукты, устанавливая `isInLibrary: true`
- **Сохраняет оригинальные ID продуктов** (чтобы не сломать ссылки в других коллекциях)

**Безопасность:**
- Использует `set()` с `merge: true` для сохранения всех существующих полей
- Сохраняет ID документов, поэтому ссылки в `InventoryLine`, `PurchaseOrderLine` и других коллекциях продолжают работать

### 3. Миграция продуктов (`seed-library-products-and-premixes.ts`)

Скрипт для первоначальной миграции продуктов и создания премиксов.

**Запуск:**
```bash
npx tsx scripts/seed-library-products-and-premixes.ts
```

## Порядок выполнения

1. **Сначала запустите диагностику:**
   ```bash
   npx tsx scripts/diagnose-products.ts
   ```
   Это покажет текущее состояние базы данных.

2. **Затем запустите копирование:**
   ```bash
   npx tsx scripts/copy-old-products-to-library.ts
   ```
   Это скопирует старые продукты в библиотеку.

3. **Проверьте результаты:**
   - Запустите диагностику снова, чтобы убедиться, что продукты скопированы
   - Проверьте приложение - продукты должны появиться в библиотеке

## Устранение проблем

### Ошибка: "Unable to detect a Project Id"

**Причина:** Не настроены учетные данные Firebase Admin.

**Решение:**
1. Убедитесь, что файл `serviceAccountKey.json` находится в корне проекта
2. Или установите переменную окружения `FIREBASE_SERVICE_ACCOUNT_KEY`
3. Проверьте, что JSON-файл валидный

### Библиотека пуста после выполнения скрипта

**Возможные причины:**
1. Все продукты имеют `barId` (персональные) - они не копируются в библиотеку
2. Все продукты уже имеют `isInLibrary: true`
3. Ошибка при выполнении скрипта (проверьте логи)

**Решение:**
1. Запустите диагностику, чтобы увидеть реальное состояние
2. Проверьте логи выполнения скрипта
3. Убедитесь, что в приложении правильный запрос для библиотеки

### Продукты не отображаются в приложении

**Проверьте:**
1. Запрос в `src/contexts/products-context.tsx` - должен быть `where('isInLibrary', '==', true)`
2. Фильтрация в `libraryProductsList` - должна исключать продукты с `barId`
3. Кэш браузера - попробуйте очистить `localStorage` или обновить страницу
